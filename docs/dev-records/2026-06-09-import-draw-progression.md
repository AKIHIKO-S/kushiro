## 3. ダブルス取込の精度改善（横並び誤読＋結合セル過大計上） — `e7de881`

**背景**: 全15サンプルでの取込回帰を実施し、ダブルスで2系統の誤読を発見。
特に `なごやか亭2021 男子ダブルス` が **196件（実数の2倍）** に膨れ、相方が半分欠落していた。

**原因と対応**（`tools/parse_bracket_seedlist.js`）:
1. **横並び誤読**: `seed|名1|名2|所属1|所属2`（所属カッコ無し）が縦ペア判定に誤落ちし、
   相方を所属フィールドに取違え＋連続2行を1ペアに誤結合していた。
   → 「**直下のseed列に番号があるか**」で横/縦を確実に分離
2. **結合配置セルの過大計上**: ブラケット配置セルが `大野・馬場（団体）` のように2姓を1セルに結合表記し、
   別載の分割シードリストと併存して二重計上。
   → 分割ペアが在るとき、結合単独セル（相方なし or 相方欄が所属断片）を冗長として除去。
     分割が皆無なら結合側を正本として温存（安全弁）

**効果（相方欠落=0達成）**:

| ファイル | 修正前 | 修正後 |
|---|---|---|
| なごやか亭2021 男子D | 196（欠落98・誤読） | **98（欠落0・全分割正確）** |
| ヤサカ杯 混合D / 男子D | 33 / 48（欠落12,20） | **17 / 24（欠落0）** |
| (修正)01ヤサカ一般（実運用） | 完璧 | **不変** |

**品質ゲート**: 既存13サンプルは**行レベルで完全同一＝無回帰**（バイト一致で確認）。
回帰テスト `test/seedlist-doubles-layout.test.js` 追加（横/縦/カッコ付き/結合除去/結合のみ温存）。

**既知の残課題（範囲外）**: `ニッタク杯_記録用.xlsx` の重複は「記録用（試合記録）」エクスポート由来の別物。

## 7. 当日進行（勝者進出）の検証＋回帰固定 — `6a7dad9`

**背景**: 進行の核心（`finishMatchInternal`/`advanceWinnerInline`/`correctResult`）は
冪等性・競合検知・Elo整合性・原子性まで作り込まれた成熟コードだが、
「**勝者の次戦進出そのもの**」に専用テストが無く（elo-integrity/team-tie はElo・団体内訳のみ）、
壊れるとブラケットが静かに不整合化するリスクがあった。

**対応**（`test/progression-advance.test.js` 新設、5件。全て一発合格＝ロジック健全）:
- finish→勝者が `next_match` の正しいスロットへ進出
- BYE は自動繰り上げ（generateBracket 時に実選手が次戦へ）
- 冪等: 同じ勝者の再finishで二重進出しない（連打/オフライン再送）
- 競合: 完了済みを別勝者で finish → 上書きせず `conflict`
- 修正: `correctResult` で勝者反転 → 次戦が新勝者へ貼り替わる

## 技術メモ（再利用のための要点）

- **取込パーサの横/縦判定**: ダブルスのレイアウト識別は「直下のseed列に番号があるか」が最も頑健な信号。
  番号あり=1行1ペア(横)、なし=2行1ペア(縦)。所属カッコの有無に依存しない。
- **結合セル除去の安全弁**: 「分割ペアが1件でも在るときだけ結合単独セルを落とす」。
  結合表記しか無いファイルは正本として温存し、データ消失を防ぐ。
- **as_drawn は全選手に組番号(seed≥1)が必要**。パーサは常に 1..N を付与するため実フローでは問題なし。
- **抽選の分離品質**: 分離可能ならR1同所属0、分離不能でも `max(0, B−size/2)` の理論最小。
  most-constrained-first 配置 ＋ R1同所属の swap 修復で達成。再現性は draw_seed と id 全順序化で担保。
- **進行の安全性**: finish は冪等ガード（同一勝者は無害）＋競合検知（別勝者は上書きせず conflict）。
  結果変更は必ず `correctResult` 経由（次戦クリア→再進出を1トランザクション）。
- **PII対策**: 実大会ファイルは commit 不可。回帰は合成データ（PII-free）でパイプライン全体を固定する方針。
