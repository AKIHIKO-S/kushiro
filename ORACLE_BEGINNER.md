# Oracle Cloud 完全初心者ガイド — KTTA Platform

**ドメイン不要・お名前.com 不要・完全無料** で KTTA Platform を運用する手順。

このガイドは「ターミナルなんて触ったことない」レベルの方を想定しています。
**コピペできるところは全てコピペ用にしています。** 分からない時は「⚠ よくあるトラブル」を見てください。

所要時間: **約 2 時間** (初回のみ・休憩込み)

---

## このガイドで使う「ニックネーム」

| 用語 | 意味 |
|---|---|
| **インスタンス** | レンタルしたサーバー (パソコン1台と思ってください) |
| **SSH** | サーバーに「鍵」を使って中に入る方法 |
| **ターミナル / コンソール** | コマンドを打つ黒い画面 |
| **公開IPアドレス** | サーバーの「住所」(例: 132.226.45.78) |
| **nip.io** | アドレスを自動で URL にしてくれる無料サービス |
| **HTTPS** | アドレスの先頭が `https://` で始まる安全な接続 |

# 【第1部】 Oracle Cloud アカウント作成 (30分)

## 1-1. サインアップ画面を開く

1. ブラウザで以下を開く:
   ```
   https://www.oracle.com/jp/cloud/free/
   ```
2. 画面中央の **「無料で始める」** または **「Always Free」** ボタンをクリック

## 1-2. アカウント情報を入力

1. **アカウント名** に好きな名前 (例: `ktta-kushiro`) を入力
   - これは Oracle Cloud 全体での識別名なので、世界で1個だけ
   - 既に使われていたら違う名前にする
2. **国/地域**: `Japan` を選択
3. **名・姓**: アルファベットで入力 (例: `Akihiko` / `Sato`)
4. **メールアドレス** を入力
5. メールに認証コードが届く → そのコードを入力

## 1-3. パスワード設定

8文字以上・大文字・小文字・数字・記号 を混ぜたパスワード。
忘れないようメモを推奨。

例: `Ktta@Kushiro2026!` (これは例なので、自分のを作る)

## 1-4. ⚠ 超重要: ホームリージョンの選択

**この画面は一度きり・変更不可!**

- **「Home Region」のドロップダウン** → **「Japan East (Tokyo)」** を選択
- もし Tokyo が混雑で選べなければ **「Japan Central (Osaka)」**

これでサーバーが日本国内に作られ、応答が速くなります。

## 1-5. 住所・電話番号

- 住所: アルファベット表記で OK (例: `1-2-3 Yamada-cho`, `Kushiro-shi`, `Hokkaido`)
- 郵便番号: 自分のもの (例: `085-0000`)
- 電話番号: 自分の携帯
- 「電話番号確認」 → SMS が届く → コードを入力

## 1-6. 支払い情報 (請求はされません)

クレジットカード情報を入力します。これは **本人確認のため** だけです。

> 💡 Oracle Cloud Free を使い続ける限り、課金されることはありません。
> 心配なら「Always Free」リソースだけ作るようにし、有料アップグレードボタンを押さないようにします。

## 1-7. 利用規約に同意 → 完了

「同意してアカウント開始」をクリック。

**ダッシュボード**が表示されたら成功! 次に進みます。

> ⚠ よくあるトラブル
> - **「カードが拒否されました」** → 別のカード or プリペイドカード。VISA/Master/JCB すべて対応
> - **「アカウント作成中」のまま動かない** → 5〜10分待つ。それでも動かなければ別ブラウザで再ログイン
> - **「リージョンが選べない」** → 別の時間帯にもう一度試す

# 【第2部】 SSH キーの作成 (10分)

「SSH キー」= サーバーに入るための鍵。これがないとサーバーを作れません。

## 2-1. ターミナルを開く

- **Mac**: Spotlight (Cmd+Space) → 「ターミナル」と入力 → Enter
- **Windows**: スタートメニュー → 「PowerShell」 と入力 → Enter

## 2-2. 鍵を作るコマンド (コピペ)

下のコマンドを **そのまま** コピーして、ターミナルに貼付け → Enter

```bash
ssh-keygen -t ed25519 -f ~/.ssh/ktta_oracle -N ""
```

すると以下のような表示が出ます:

```
Generating public/private ed25519 key pair.
Your identification has been saved in /Users/xxx/.ssh/ktta_oracle
Your public key has been saved in /Users/xxx/.ssh/ktta_oracle.pub
...
```

これで鍵が2つできました:
- `ktta_oracle` = **秘密鍵** (絶対に人に見せない・サーバーで使う)
- `ktta_oracle.pub` = **公開鍵** (Oracle に登録する)

## 2-3. 公開鍵の中身を表示

下のコマンドをコピペ → Enter:

```bash
cat ~/.ssh/ktta_oracle.pub
```

すると一行のテキストが表示されます (例):

```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIH... ktta@kushiro
```

**この一行を後で Oracle Cloud に貼付けます。** ターミナルは閉じずに残してください。

> ⚠ よくあるトラブル
> - **`ssh-keygen` が見つからない** (Windowsのみ) → 設定 → アプリ → オプション機能 → 「OpenSSH クライアント」 を追加
> - **「既存ファイルを上書きしますか?」と聞かれた** → `y` → Enter (新しい鍵で進める)

# 【第3部】 サーバー (インスタンス) を作る (20分)

## 3-1. Oracle Cloud Console を開く

サインアップ完了後の画面、または:
```
https://cloud.oracle.com/
```

## 3-2. インスタンス作成画面へ

画面左上の **「ハンバーガーメニュー」 (☰)** をクリック → メニュー展開
1. **「コンピュート」** をクリック
2. **「インスタンス」** をクリック

「インスタンス」一覧画面が出る (最初は空っぽ)

**「インスタンスの作成」** ボタンをクリック (青いボタン)

## 3-3. インスタンスの設定

たくさん項目がありますが、変更するのは以下だけです。

### ① 名前
- **名前**: `ktta-platform` と入力

### ② イメージとシェイプ
**「シェイプとイメージを変更」** をクリック

#### イメージ (OS)
- 「**プラットフォームイメージ**」タブが選ばれていることを確認
- 一覧から **「Canonical Ubuntu」** を選択
- バージョン: **22.04** (一番上にあるはず)
- **「イメージの選択」** をクリック

#### シェイプ (サーバー性能)
画面下の **「シェイプ」セクション** で **「シェイプの変更」** をクリック
- **「Ampere」タブ** を選択 (ARM系の永久無料枠)
- **「VM.Standard.A1.Flex」** を選択
- 下にスライダーが出る:
  - **OCPU数**: **2** にする
  - **メモリの量 (GB)**: **12** にする
- **「シェイプの選択」** をクリック

### ③ プライマリVNIC情報
- そのまま (デフォルトのまま OK)
- **「パブリック IPv4 アドレスの割当て」** にチェックが入っていることだけ確認

### ④ SSH キーの追加
- **「SSH キーの貼付け」** を選択 (デフォルトで「キーペアの生成」になっているので変更)
- 大きなテキストボックスに、**第2部で表示した公開鍵 (ssh-ed25519 で始まる一行)** を貼付け

### ⑤ ブート・ボリューム
- そのまま (デフォルト 50GB)

## 3-4. 「作成」ボタンを押す

画面下の **「作成」** をクリック。

「プロビジョニング中...」と表示され、約 **2〜5分** で完了。

完了すると **「実行中」 (緑色)** に変わります。

## 3-5. 公開 IP アドレスを記録

インスタンス詳細画面の右下に **「インスタンスのアクセス」セクション** があります。

```
パブリックIPアドレス: 132.226.45.78    [コピー]
```

この数字 (4組の数字) を**メモ帳に必ずコピーして保存**してください。

以下、このIPアドレスを `<YOUR_IP>` と表記します。あなたの場合は具体的な数字に置き換えてください。

> ⚠ よくあるトラブル
> - **「Out of host capacity」エラー** → 数時間後にリトライ。それでも駄目なら Osaka リージョンへ
> - **OCPU が選択できない** → Free Tier の上限到達。OCPU=1 / メモリ=6 で試す
> - **公開IPが表示されない** → ✅ 「パブリックIPv4の割当て」にチェックを入れ忘れている。インスタンス削除して作り直し

# 【第4部】 ファイアウォール開放 (10分)

サーバーができても、外からアクセスできる「穴」が開いていないとブラウザで開けません。
80番ポート (HTTP) と 443番ポート (HTTPS) を開放します。

## 4-1. VCN (仮想ネットワーク) を開く

Oracle Cloud Console で:
1. ☰ メニュー → **「ネットワーキング」** → **「仮想クラウド・ネットワーク」**
2. 一覧に出てくる VCN (例: `vcn-20260526-1234`) をクリック

## 4-2. セキュリティ・リストを開く

VCN詳細画面 → 左下のリソース一覧:
- **「セキュリティ・リスト」** をクリック
- **「Default Security List for vcn-...」** をクリック

## 4-3. 受信ルールを追加

「イングレス・ルール」セクションで **「イングレス・ルールの追加」** をクリック

### ルール1: HTTP (ポート80)
- **ステートレス**: チェックしない
- **ソース・タイプ**: `CIDR`
- **ソースCIDR**: `0.0.0.0/0`
- **IPプロトコル**: `TCP`
- **ソース・ポート範囲**: (空欄のまま)
- **宛先ポート範囲**: `80`
- **説明**: `HTTP`

下の **「別のイングレス・ルール」 (+)** をクリック

### ルール2: HTTPS (ポート443)
- **ソース・タイプ**: `CIDR`
- **ソースCIDR**: `0.0.0.0/0`
- **IPプロトコル**: `TCP`
- **宛先ポート範囲**: `443`
- **説明**: `HTTPS`

**「イングレス・ルールの追加」** で保存

> ⚠ よくあるトラブル
> - **保存後もブラウザでつながらない** → 第5部の OS 内側のファイアウォール開放も必要。安心してください、後で自動でやります

# 【第5部】 サーバーに SSH 接続 (10分)

## 5-1. ターミナルを開く

(第2部で開いたままでも OK)

## 5-2. 接続コマンド

下のコマンドの `<YOUR_IP>` を、**第3部でメモした実際のIPアドレス** に置き換えて実行:

```bash
ssh -i ~/.ssh/ktta_oracle ubuntu@<YOUR_IP>
```

例: IPが `132.226.45.78` なら:
```bash
ssh -i ~/.ssh/ktta_oracle ubuntu@132.226.45.78
```

## 5-3. 初回接続の確認

初めて接続するときに以下が表示されます:

```
The authenticity of host '132.226.45.78' can't be established.
ED25519 key fingerprint is SHA256:xxxxxxxx
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

→ `yes` と入力 → Enter

## 5-4. 接続成功

サーバーに入れると、以下のようなプロンプトに変わります:

```
ubuntu@ktta-platform:~$
```

これが **「サーバーの中にいる」** 状態です。
これ以降のコマンドは **すべてサーバー上で実行** されます。

> ⚠ よくあるトラブル
> - **`Permission denied (publickey)`** → SSH キーが違う。`-i ~/.ssh/ktta_oracle` の部分が正しいか確認
> - **`Connection refused` または `timed out`** → 数分待ってから再試行。インスタンスがまだ起動中の可能性
> - **`No such file or directory`** → `~/.ssh/ktta_oracle` がない。第2部からやり直し

# 【第6部】 KTTA Platform のインストール (15分)

サーバー (ubuntu@ktta-platform:~$) にいる状態で、以下のコマンドを **1行ずつ** コピペします。

## 6-1. Git をインストール

```bash
sudo apt update
```

途中で `[Y/n]` と聞かれたら `y` Enter (もし聞かれなければそのまま待つ)

```bash
sudo apt install -y git
```

## 6-2. KTTA Platform をダウンロード

```bash
git clone https://github.com/AKIHIKO-S/kushiro.git /tmp/ktta
sudo mv /tmp/ktta /opt/ktta
cd /opt/ktta
```

## 6-3. インストールスクリプトを実行

ここが一番重要。下のコマンドを **そのまま** 実行:

```bash
sudo ./deploy/install.sh $(curl -s ifconfig.me).nip.io
```

これは何をしているか:
- `curl -s ifconfig.me` → サーバーの公開IPを取得 (例: 132.226.45.78)
- `.nip.io` を末尾につける → `132-226-45-78.nip.io` という「無料の即席アドレス」を作る
- そのアドレスで `install.sh` を実行

**3〜10分** かかります。コーヒーブレイク☕

完了すると下のような表示で終わります:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  セットアップ完了!
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  管理画面:    https://132-226-45-78.nip.io/admin/
  観戦ビュー:  https://132-226-45-78.nip.io/viewer/live/
  管理キー:    /etc/ktta.env を確認 (ADMIN_KEY=abc123...)
```

## 6-4. 管理キーをメモ

下のコマンドで管理キーを表示:

```bash
sudo cat /etc/ktta.env | grep ADMIN_KEY
```

下のような出力:

```
ADMIN_KEY=a3f7c2e89b1d4e567890abc123def456
```

`=` の右側の文字列をメモ帳にコピーしてください。

> ⚠ よくあるトラブル
> - **npm install で「killed」エラー** → メモリ不足。OCPU=2/12GB なら起きないはず。1OCPU/6GB なら swap 設定が必要
> - **certbot がエラー** → ifconfig.me が応答しなかった可能性。後で手動で `sudo certbot --nginx -d <IP>-nip.io` を実行
> - **「sudo: command not found」** → ubuntu 以外のユーザーで実行している。`ssh ubuntu@<IP>` で再ログイン

# 【第8部】 GitHub Actions 自動デプロイ (任意・10分)

これを設定すると、ローカル PC で修正 → `git push` するだけで自動で本番反映されます。

## 8-1. デプロイ用 SSH 鍵を作る

サーバー上で:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/github_deploy -N ""
cat ~/.ssh/github_deploy.pub >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

## 8-2. 秘密鍵を表示

```bash
cat ~/.ssh/github_deploy
```

`-----BEGIN OPENSSH PRIVATE KEY-----` から `-----END OPENSSH PRIVATE KEY-----` までを **全部** コピー。

## 8-3. GitHub Secrets に登録

ブラウザで:
1. https://github.com/AKIHIKO-S/kushiro/settings/secrets/actions
2. **「New repository secret」** をクリック
3. 以下を順番に追加:

### Secret 1
- **Name**: `ORACLE_HOST`
- **Secret**: あなたの公開IP (例: `132.226.45.78`)

### Secret 2
- **Name**: `ORACLE_SSH_KEY`
- **Secret**: 8-2でコピーした秘密鍵全部

`Add secret` をクリック

## 8-4. テスト

ローカル PC のターミナルで:

cd ~/Desktop/claude/tabletennis

# トラブル時 困ったらここを見る

## サイトが繋がらない

1. **インスタンスは動いているか?**
   - Oracle Cloud Console → コンピュート → インスタンス
   - 「実行中」になっているか

2. **ファイアウォールは開いているか?**
   - VCN → セキュリティ・リスト → 80/443 のルールがあるか

3. **アプリは動いているか?**
   - SSH で接続 → `sudo systemctl status ktta`
   - 「active (running)」と緑色で表示されていればOK
   - そうでなければ: `sudo systemctl restart ktta`

## 「Bad Gateway」エラー
- Node.js プロセスがクラッシュ
- `sudo journalctl -u ktta -n 50` でエラー確認
- `sudo systemctl restart ktta` で復旧

## 何もかも分からなくなった
- まずスクリーンショットを撮る
- インスタンスを止めずに「困った状態」のまま聞く
- インスタンスを削除しない (中のデータが消える)
