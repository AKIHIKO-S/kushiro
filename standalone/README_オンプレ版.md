# KTTA Platform — オンプレ版（他PCにインストールして手元で動かす）

クラウド（Oracle）を使わず、**会場のノートPC1台**でシステムを動かす方法です。
インターネットが無くても、進行管理・観戦ビュー・トーナメント表・選手DBはすべて動きます
（GoogleフォームによるWeb申込・GAS集計だけはインターネットが必要）。

データはすべて **そのPCの `data/` フォルダ（SQLite）** に保存されます。

---

## 1. 必要なもの（1回だけ）

- **Node.js（LTS版）** … https://nodejs.org/ja からダウンロードしてインストール
  - インストール確認: ターミナル/コマンドプロンプトで `node -v` と打って `v22.x` 等が出ればOK
- このアプリ一式（フォルダごとコピー）

> オフライン会場で使う場合は、ネットがある場所で一度 `npm install --omit=dev` を実行し、
> `node_modules` フォルダごとUSB等でコピーして持ち込んでください（初回のネット接続不要に）。

---

## 2. 起動（毎回）

### macOS
`standalone/start.command` を **ダブルクリック**。
（初回は「開発元が未確認」と出たら、右クリック→「開く」で許可）

### Windows
`standalone/start.bat` を **ダブルクリック**。

初回起動時に **管理キー** が自動生成され画面に表示されます（`.env.local` に保存）。
管理画面 `/admin` を開いたら、最初にこのキーを入力してください。

起動すると自動でブラウザが開きます:
- 管理画面: `http://localhost:3000/admin/`
- 観戦ビュー: `http://localhost:3000/viewer/`
- 大画面（プロジェクター用）: `http://localhost:3000/viewer/live/`

終了するには、黒い画面（ターミナル）で **Ctrl + C**、またはウィンドウを閉じます。

---

## 3. 会場の他の端末（スマホ・別PC）から見る

同じWi-Fi/LANに繋がっていれば、起動時に表示される
`http://（このPCのIPアドレス）:3000/viewer/live/` を他端末のブラウザに入力すれば閲覧できます。

- 観戦者・選手にはこのURL（またはQRコード：観戦ビューの「共有」ボタン）を案内してください。
- WindowsのファイアウォールでNode.jsの通信を「許可」する必要がある場合があります。

> プッシュ通知（マイ番号の呼出通知）は **https** が必要なため、localhost以外（IP直打ち）では
> 通知が制限されます。会場通知を確実に使いたい場合はクラウド（https）運用を推奨します。
> ページを開いている間の画面内通知・音・バイブはローカルでも動作します。

---

## 4. バックアップ（重要）

大会データは `data/tournament.db` にあります。大会後はこのファイルをコピーして保管してください。
（`data/` フォルダごとコピーすればOK。USBやクラウドドライブへ）

付属のスクリプト `npm run backup` でも取得できます（`backups/` に保存）。

## 5. 「インストーラ（パッケージ版 .exe / .app）」は作れる？

**作れます。** ただし本アプリは高速DBに **better-sqlite3（ネイティブ拡張）** を使うため、
ワンクリック実行ファイル化には少し準備が必要です。主な選択肢:

| 方法 | 成果物 | long点 / 注意 |
|------|--------|-----------|
| **A. start スクリプト（本フォルダ）** | フォルダ配布 | 最も簡単・確実。Node.jsの事前インストールだけ必要。**まずはこれを推奨。** |
| **B. pkg** | 単一 `.exe` / バイナリ | Nodeのインストール不要。ただし `better-sqlite3` の `.node` を同梱し、**ビルドは配布先と同じOS/CPUで行う**必要あり。 |
| **C. Electron** | デスクトップアプリ（.exe/.dmg、ウィンドウ付き） | 最も「アプリらしい」。容量大・ビルド構成が複雑。 |

### B. pkg での単一EXE化（上級者向け・手順の概要）
配布したいPCと同じOS（例: Windows）で:
```bash
npm i -D pkg
# better-sqlite3 のネイティブバイナリを同梱する設定が必要
npx pkg . --targets node22-win-x64 --output ktta.exe \
  --assets "node_modules/better-sqlite3/build/Release/*.node"
```
- `package.json` に `"bin": "server.js"` と `pkg.assets` を追加します。
- 生成した `ktta.exe` と `public/`, `data/`（空でOK）を同じフォルダに置いて実行します。
- **ネイティブ拡張のため、Mac で作った exe は Windows では動きません。** 必ず対象OSでビルドしてください。

> 当面は **A（startスクリプト）** が最も安全・確実です。Bの完全自動ビルドが必要なら、
> 対象PC（Windows等）で実際にビルド・動作確認しながら進めるのが安全です。

## 6. よくある質問

- **Q. ポート3000が使われている**: 起動前に環境変数 `PORT` を変えてください（例 `PORT=8080`）。
- **Q. データを別PCへ移したい**: `data/` フォルダをコピーして同じ場所に置けばそのまま使えます。
- **Q. 管理キーを忘れた**: `.env.local` を開けば書いてあります。変更したい場合はファイルを編集して再起動。
