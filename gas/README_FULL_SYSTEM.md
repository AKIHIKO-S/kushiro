# 釧路卓球協会 大会申込・集計・トーナメント支援システム (GAS)

KTTA Platform 本体（運営システム）とは独立した、Google スプレッドシート + Apps Script ベースの「申込〜集計〜トーナメント表作成」までを完結する補助ツールです。

## 構成

```
申込フォーム (GAS Web App)
   ↓ POST
スプレッドシート (このシステム)
   ・申込台帳
   ・選手名簿
   ・団体戦 / ダブルス / ミックス / シングルス
   ・お弁当・懇親会
   ・ダブルス相手募集
   ・団体別集計 (人数+参加料)
   ・種目別一覧表
   ・トーナメント表 (自動配置)
   ↓ Excel 書出 (KTTA Platform 取込テンプレ形式)
KTTA Platform (Render)
   ・大会運営 (試合呼出/結果入力/観戦ビュー)
```

## セットアップ手順

### 1. スプレッドシートを新規作成
[Google スプレッドシート](https://sheets.new) で新規ファイルを作成します。
名前は「**◯◯杯 申込システム**」のように分かりやすく付けます。

### 2. Apps Script を開く
- メニュー「拡張機能」→「Apps Script」
- 開いた画面のデフォルトの `function myFunction() {}` を全て削除
- このリポジトリの `gas/ktta_full_system.gs` の全内容を貼り付け
- 上部の「保存」アイコン (💾) をクリック

### 3. スクリプトプロパティを設定（任意）
- プロジェクト設定 (歯車アイコン)
- 「スクリプト プロパティ」セクション
- 「スクリプト プロパティを追加」で以下を設定:

| キー | 値 |
|---|---|
| `ASSOCIATION_NAME` | 釧路卓球協会 |
| `PRESIDENT_NAME` | 会長 山本 満 |
| `VENUE` | ウインドヒルくしろスーパーアリーナ |
| `ADMIN_EMAIL` | 管理者メールアドレス (BCC 用) |
| `PRICE_TEAM_M` | 4000 |
| `PRICE_TEAM_F` | 4000 |
| `PRICE_DBL_M` | 1000 |
| `PRICE_DBL_F` | 1000 |
| `PRICE_MIX` | 1200 |
| `PRICE_SGL_M` | 700 |
| `PRICE_SGL_F` | 700 |
| `PRICE_BENTO` | 800 |
| `PRICE_PARTY` | 3500 |

設定しなくてもデフォルト値で動きます。

### 4. ウェブアプリとしてデプロイ
- 右上の「デプロイ」→「**新しいデプロイ**」
- 種類: **ウェブアプリ**
- 説明: 「KTTA 申込フォーム v1」など
- **次のユーザーとして実行**: 自分
- **アクセスできるユーザー**: 全員
- 「デプロイ」→ 表示された **ウェブアプリ URL をコピー** (これを Jimdo に貼り付け)

### 5. スプレッドシートに戻る
ブラウザで F5 (再読込) すると、メニューバーに「**KTTA**」が追加されています。

### 6. 初期セットアップ実行
KTTA メニュー →「**初期セットアップ (全シート作成)**」
これで 12 個のシートが自動で作成されます。

### 7. 申込フォーム URL を共有
KTTA メニュー →「**申込フォームの URL を表示**」
表示された URL を Jimdo の HTML 埋込ウィジェットに貼り付け:

```html
<iframe src="ここに URL を貼る"
        width="100%" height="1400"
        style="border:0;display:block;width:100%"
        title="大会申込フォーム"
        loading="lazy"></iframe>
```

## 運用フロー

### 申込期間中
- 申込者が Jimdo のフォームから送信
- 自動で 8 シートに振り分け転記
- 管理者には確認メールが BCC 送信される
- 申込者には自動返信メール
- KTTA メニュー →「団体別集計を再計算」で参加料合計が更新

### 申込締切後
1. KTTA →「**団体別集計を再計算**」
2. KTTA →「**種目別一覧を再構築**」
3. KTTA →「**領収書 PDF を一括生成**」(Google Doc + PDF 生成)
4. KTTA →「**トーナメント表シートを作成**」(種目ごとに)
5. KTTA →「**KTTA Platform 用 Excel を書出**」
   → スプレッドシート全体を **ファイル → ダウンロード → Excel (.xlsx)** で書出
   → KTTA Platform の「Excel / PDF 取込」からアップロード

### 大会当日
- KTTA Platform の「進行管理」タブで運営
  - 試合呼出 / 台割当 / 審判アサイン / 結果入力
  - 観戦ビューを観客に公開 (QR コード)
- このシステム (GAS) は不要

## メニュー機能一覧

| メニュー項目 | 内容 |
|---|---|
| 初期セットアップ | 全シートを作成・ヘッダー設定 |
| 申込フォームの URL を表示 | デプロイ済みフォーム URL を確認 |
| テスト申込を投入 | 動作確認用のサンプル申込 |
| 団体別集計を再計算 | 申込台帳から団体ごとの人数+料金を再構築 |
| 種目別一覧を再構築 | 種目ごとの選手リストを作成 |
| 領収書 PDF を一括生成 | 団体ごとに領収書を Google Doc → PDF 化 |
| トーナメント表シートを作成 | 種目を指定 → シード配置のブラケットを自動レイアウト |
| KTTA Platform 用 Excel を書出 | 案内表示 (Excel 書出は標準機能) |
| 設定を編集 | スクリプトプロパティ を画面で編集 |

## トラブルシューティング

### 「フォーム送信時にエラーになる」
- デプロイ済みかチェック (デプロイ → 「デプロイの管理」)
- ウェブアプリ URL を再確認
- 「アクセスできるユーザー: 全員」になっているか確認

### 「メニューに KTTA が表示されない」
- ブラウザで F5 リロード
- ファイル → 「再読込」

### 「集計が更新されない」
- KTTA →「団体別集計を再計算」を手動実行
- スクリプトの実行権限が許可されているか確認

### 「領収書 PDF が空白」
- 集計が完了しているか確認 (先に「団体別集計を再計算」を実行)
- 各団体に少なくとも 1 名の申込があるか確認

## アンインストール / リセット

- スプレッドシートを削除すれば、全データが削除される
- Apps Script のデプロイを停止: デプロイ → 「デプロイの管理」 → 「アーカイブ」

## ライセンス

このスクリプトは KTTA Platform プロジェクトの一部です。釧路卓球協会の運営目的に自由に使用できます。
